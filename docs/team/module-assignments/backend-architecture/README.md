# Backend Architecture Module

## Directory Structure

*   **`01-work-description.md`**: The complete scope of this module.
*   **`02-phase-1-research/`**: Phase 1 specific assignments and overview.
*   **`03-phase-2-prd+plan/`**: Phase 2 specific assignments and overview.

### Working Directories

*   **`deliverables/`**: **Finalized, reviewed work.** Look here for the official "truth" of the module (PRDs, Specs, Research Briefs).
*   **`work-in-progress/`**: **Active construction zone.**
    *   `raw/`: Unprocessed notes and transcripts.
    *   `ai-generated/`: AI outputs needing validation.
    *   `synthesis/`: Manual refinements and drafts.
*   **`handoffs/`**: **Integration packages.** Code and docs ready for other developers to use.

---
**Note**: Do not create files at this root level. Use the appropriate subdirectory.
