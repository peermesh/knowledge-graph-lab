## 🎯 Design Principles

**Every file in this structure follows these rules:**

- **One clear purpose** - Each file answers a specific question
- **One clear next step** - Every file points to what to do next  
- **No jargon without explanation** - Written for CS students, not AI experts
- **Workbook style** - Tell interns exactly what to do, no mystery solving
- **Minimal complexity** - Only essential files, no extra navigation layers

**Week 1**: Research phase - technology evaluation and planning

**Week 3**: Project setup - development environment and tooling  
**Weeks 4-8**: Development - core functionality through advanced features  
**Week 9**: Documentation and review - preparing for demo  
**Week 10**: Completion - demo day and retrospective
